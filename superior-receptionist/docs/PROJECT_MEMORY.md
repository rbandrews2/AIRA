# AIRA AI Project Memory

## Product Intent

AIRA AI is currently aimed at acting as a call assistant with secretary-like duties for Superior Consultation, LLC.

The long-term plan is to develop AIRA into a full AI smart phone operating system. That future version may include native Android/iOS components, device-level administration, carrier integrations, and on-device AI.

## Permission Philosophy

The app should request powerful permissions transparently during installation or onboarding and those permissions must be granted by the user or phone operator.

For the PWA:

- Request browser-supported permissions directly, such as camera/video and microphone.
- Explain permissions the PWA cannot grant.
- Provide a clear stop-install path.
- Do not imply that a web app can silently receive root, super-user, overlay, system settings, call control, SMS, or device admin access.

For future native builds:

- Use legitimate Android and iOS permission flows.
- Use Android roles and settings screens for capabilities like call screening, overlay, write settings, and device admin.
- Use iOS CallKit only for supported VoIP workflows.
- Use telephony-provider or carrier integrations for business-number answering, transfers, hold music, SMS, and call recording.
